Мораль

  • нет «плохих» браузеров;
  • читайте спецификации;
  • читайте публикации сообщества (например, Flexbugs).

2. Поймать

Как поймать тигра в клетке?

Отлов бага

  1. Отрезать часть.
  2. Проверить.
  3. Перейти к пункту 1.

Проверяем баг в «ночных» сборках

  • Firefox Nightly;
  • Chrome Canary;
  • Opera Developer;
  • Safari Technology Preview;
  • «ночные» сборки Edge.
«Ночные» сборки могут содержать новые баги!

Проверяем баг в других системах

  • сервисы: Browserstack, Saucelabs;
  • виртуальные машины с Windows;
  • Remote Desktop: EdgeHTML на Windows 10 для Mac.

3. Сообщить


Опрос в Твиттере

Опрос «The Ultimate CSS Survey»

Можно отправить репорт из браузера

  • Меню > Справка > Сообщить о проблеме;
  • есть в Chrome, Opera, Firefox.

Что включить в репорт?

  • версия браузера и операционной системы;
  • как воспроизвести;
  • что я ожидал;
  • что я получил.

Ищем баг в трекере

  • Firefox Bugzilla;
  • Chromium bugs;
  • Opera bug report wizard;
  • WebKit Bugzilla;
  • Internet Explorer и Edge;

Webcompat

4. Обойти

После репорта

  • отслеживайте судьбу багрепорта;
  • придумайте «хак»;
  • попробуйте его автоматизировать.

Автоматизация «хаков»

  • PostCSS Flexbugs Fixes — автоисправление багов Flexbugs;
  • PostCSS Gradient Transparency Fix — фикс бага с transparent в линейных градиентах в Safari.

Привлеките внимание общественности

  • в твиттере, гитхабе, слаке;
  • в публикациях;
  • голосуйте за решение багов.

Опишите проблему подробно, предложите обходной путь.

Лайфхак

  • обновляйтесь по понедельникам;
  • репортите баги по пятницам 🤓.
